# 2026-06-27

## client-workflow — shipped

Implemented the full client engagement workflow system across backend, frontend, and E2E tests.

### What was built

**Backend (Spring Boot)**
- V13 migration: added `business_type` (PERSONAL/CORPORATE/SELF_EMPLOYED) and `fiscal_year_end_month`/`fiscal_year_end_day` to `clients`. PERSONAL always stores Dec 31 server-side; CORPORATE/SELF_EMPLOYED require a valid calendar date validated with `MonthDay.of()`.
- V14 migration: `client_engagements` (UNIQUE on client_id + tax_year) and `client_engagement_history` (full audit trail with nullable `from_status`).
- Five-state machine — any-to-any transitions; no guard logic needed.
- Bilingual (EN/ZH) email notifications on IN_PROCESSING, PENDING_CLIENT_REVIEW, SUBMIT_TO_CRA, COMPLETED. Mail failure swallowed — status transition always succeeds.
- Cross-admin authorization enforced via `findClientForAdmin()` helper in `ClientEngagementService`. `listAll()` filters to calling admin's clients only.

**Frontend (Angular)**
- `AdminWorkflowComponent` at `/admin/workflow`: table with 6 columns, status + business-type `mat-select` filters using `computed()`.
- `AdminClientWorkflowComponent` at `/admin/clients/:id/workflow`: expandable engagement rows with lazy history fetch, "New Engagement" dialog, "Change Status" dialog with optional note.
- `AdminClientDialogComponent` updated: business type select + conditional FYE month/day inputs (hidden for PERSONAL).
- 5 Playwright E2E tests covering dashboard filters and full create → transition → history flow.

### Issues resolved

- **Security review** caught two HIGH findings before EVAL: (1) engagement endpoints didn't enforce cross-admin ownership, (2) `listAll()` exposed all admins' data. Both fixed in a single security pass.
- **Group 5 EVAL needed 2 attempts**: evaluator falsely claimed TestBed failed 0/9 (tests actually passed). Legitimate fixes applied: replaced nested subscribes with `switchMap+tap`, added `takeUntilDestroyed()` on all subscriptions, removed unused import.
- **`provideRouter([])` ordering**: must be placed before the custom `ActivatedRoute` mock or the router overrides the mock.

- Frontend: 9 new Vitest tests for client-workflow tab; 14 pre-existing file-level failures unchanged
- E2E: 5 new Playwright tests (3 dashboard, 2 client workflow)
