# Dev Log ‚Äî 2026-05-12

## 1. Backend startup & configuration fixes

**Files changed:** `start.sh`, `application.yml`, `application-dev.yml`, `application-prod.yml`, `contact/domain/ContactMessage.java`

**Issue:** `./start.sh` failed with `JavaMailSender bean not found` because no Spring profile was active, so `@Profile("dev") LoggingMailSender` never loaded. Additionally, `contact.notification-email` had no binding in the base config, causing `MailProperties @NotBlank` validation to fail at startup.

**Fixes:**
- `start.sh` ‚Äî added `export SPRING_PROFILES_ACTIVE="${SPRING_PROFILES_ACTIVE:-dev}"` so local dev always uses the `dev` profile unless overridden
- `application.yml` ‚Äî added `contact.notification-email: ${CONTACT_NOTIFICATION_EMAIL}` (required in all profiles)
- `application-dev.yml` ‚Äî replaced duplicate content with a single comment; no property overrides are needed for dev (behaviour is handled by `@Profile("dev")` beans)
- `application-prod.yml` ‚Äî added `spring.mail.*` SMTP block using env-var references; `app.cookie.secure: true` was already present
- `ContactMessage.java` ‚Äî removed leftover noise comment `// getters and setters` (flagged in group 1 code review but never committed)

**`.env` update:** added `CONTACT_NOTIFICATION_EMAIL` key (value set by user)
