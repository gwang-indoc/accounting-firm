## 4. contact-success-state — Group 1: Eliminate post-submit validation-error flash

**Commit:** `8b2ad01`

**Feature:**

- Root cause: `FormGroup.reset()` clears values and touched/dirty state but does NOT reset the `FormGroupDirective.submitted` flag. Angular Material's default `ErrorStateMatcher` shows `<mat-error>` when a control is invalid AND (touched OR submitted). After a successful send, empty required fields flashed red "Name is required" etc.
- Fix: replaced `this.form.reset()` with `this.formDirective().resetForm()`, accessed via `viewChild.required(FormGroupDirective)`. This resets values AND the directive's submitted flag, returning the form to a pristine pre-submission state.
