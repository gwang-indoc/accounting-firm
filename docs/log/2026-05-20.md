## Phase 2 — Admin Frontend

### Commits

### Features delivered

- **AdminClientsComponent extended**: Messages action button added to each client row. Unread badge (`·N`) renders only when `admin_unread_count > 0` for that client. Loads `/api/clients/unread-counts` on init and merges into row data via `unreadCounts = signal<Record<number, number>>({})`.

- **AdminClientThreadsComponent**: Thread list page at `/admin/clients/:id/messages`. Shows all threads for a client, sorted newest-first. Each row shows subject, relative timestamp, last-message preview, and an unread chip. "+ New Thread" button opens `NewThreadDialogComponent` (Material dialog). On dialog close with result, navigates to the new thread view.

- **NewThreadDialogComponent**: Material dialog with subject input (max 200) and body textarea (max 5000). Returns `{ subject, body }` on submit. Validates both fields required before submitting.

- **AdminClientThreadViewComponent**: Thread view page at `/admin/clients/:id/messages/:threadId`. Displays messages as chat bubbles — admin messages right-aligned with sky-blue background (`#38bdf8`), client messages left-aligned with dark slate background (`#1e293b`). Inline compose area at bottom. Send disabled when empty/whitespace.

- **app.routes.ts**: Two new routes added: `admin/clients/:id/messages` and `admin/clients/:id/messages/:threadId`, both protected by `authGuard + adminGuard`.
